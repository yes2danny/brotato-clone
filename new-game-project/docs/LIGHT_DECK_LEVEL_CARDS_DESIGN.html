Light deckbuilder + level-up cards

Design note for this repo (waves, shop, XP, UpgradeManager-style level-ups). For deciding whether to add the feature and how to cap scope.

1. Context: what the game already has

  • Shop (between waves): spends gold, sells persistent items / stat-style effects (ItemData, weapons placeholder).
  • Level-ups (during play, paused): pick 1 of several offers — today this reads like stat knobs (HP, move, damage, fire rate), which overlaps the shop lane.
  • Combat loop: horde waves, pickups (XP gems, gold coins), WeaponController, HealthSystem.

The design problem to solve: if level-ups are mostly “more stats,” they feel redundant next to shop items.

2. One-sentence split (two lanes)

Shop = economy + permanent baseline power (gear, passives, numbers).

Deck / cards = short-horizon combat decisions — what you do for the next few seconds, next few shots, or next trigger — not another permanent +% damage shop.

3. “Very light deck” — concrete numbers

  • Starter deck: 6–10 cards (character starter or one-time draft at run start).
  • Hard cap (v1): e.g. 12 cards max for the whole run — no 40-card deck fantasy yet.
  • Level-up offer: draw 3 cards, pick 1 (matches “pick 1” muscle memory from current upgrade UI).
  • Optional deck loop: draw pile + discard pile; when draw empties, reshuffle discard (classic “deck feel”). You can ship v0 without reshuffle (cards as charges) and add shuffle later.

4. Two v1 modes — pick one to fight scope creep

Mode A — Tactical hand (recommended first)

  • Level-up: 3 cards → pick 1 → play immediately.
  • Deck grows slowly or not at all (e.g. +1 card every 2 levels, still capped).
  • Lowest UI and balance surface.

Mode B — Run draft deck (medium scope)

  • Level-up: pick 1 → add to deck; rare “remove a card” rewards.
  • More true deckbuilder, needs more card pool discipline and removal handling.
Scope: If you are fighting creep, ship Mode A first; add drafting/removal only after the card verbs feel great.

5. How it hooks into existing systems

SystemRole with cards
XPSystem / level-up pauseSame moment: pause → overlay → pick. Content changes from “4 stat buttons” to “3 cards” (or keep 4 if you prefer).
WeaponControllerCards mostly apply windows on firing (next mag, next N seconds, after reload). This supports “gun fantasy” without every card being a new gun asset.
ShopManager / goldGold buys items. Cards should not be “second shop” — avoid selling the same stat upgrades as cards in the shop at v1.
Optional bridgeShop sometimes sells reroll tokens, or an item: “next level-up shows +1 option.” Keeps lanes related but distinct.
WavesCards naturally spike during pressure; good fit for horde tempo.

6. Card examples (good vs bad) — tailored to this project

Good — verb / window / trigger (deck lane)

  1. Burst Protocol — Play: for 6s, +fire rate; then cannot shoot for 0.7s (risk window).
  2. Magnet Shot — Play: next 8 shots slightly pull pickups (touches XP/gold pickups without permanent +radius item).
  3. Breacher Round — Play: next shot deals bonus damage to “tough” target (v1: highest HP in range).
  4. Panic Dive — Play: short move burst + one tiny iframe window (execution, not a passive stat).
  5. Overclock — Play: next reload 50% faster; for 5s you take slightly more damage (tunable).
  6. Treasure Tag — Play: next 10 kills grant +1 bonus gold (hooks to ShopManager.add_gold).
  7. Chain Coupon — Play: next projectile kill splashes a weak hit to a nearby enemy.
  8. Reroll Marker — Play: gain 1 charge of “shop reroll” consumed in shop UI (reroll fantasy without merging systems).

Medium — OK if shop does not own the same thing

  • Second Wind — one-time flat heal (not permanent max HP).
  • Armor Flare — temporary armor for a few seconds.

Bad — overlaps shop / repeats the old problem

  • Permanent +% damage, permanent +max HP, permanent +move speed as the main card line.

7. Minimum UI for v1 (deckbuilder vibe without Spire-scale UI)

  • Card face: name + one rule line + optional icon.
  • Level-up: 3-up (or 4-up) choice.
  • Optional: small “deck list” in pause menu — current cards + discard count.

Skip for v1: drag-and-drop deck editor, large rarity tiers, 50+ card synergy web, relics + potions + curses + map nodes.

8. Short example run (how it feels)

Wave 1 — level up: offers Burst Protocol, Magnet Shot, Treasure Tag. Player picks Treasure Tag to feed the shop economy.

Wave 2 — level up: picks Panic Dive for defensive execution.

Shop: spends gold on a passive that makes the build tankier — permanent lane.

Wave 3 — level up: picks Burst Protocol for a pressure spike.

Net: shop = stability and numbers; deck = spikes and timing.

9. Decision checklist

Add this if you want the core fantasy to be: “I craft combat rhythm and spike windows,” not only “I stack numbers.”

Defer or skip if you are not ready to:

  • Demote or remove overlapping stat-only level-ups from the current upgrade flow, and
  • Enforce a hard cap on deck size and card pool for v1.

10. Recommendation (this repo + your stated goals)

For gun fantasy + rerolls + light deck vibe with minimal creep:

  • Ship Mode A (tactical hand) first.
  • Cards = weapon windows + triggers + reroll charges + pickup/gold tempo.
  • Shop = items / stats / economy.
  • Gun slot unlocks can stay a milestone outside the deck at first (simpler content). If you must put unlocks in the deck, make Unlock cards very rare or you balloon art + tuning.
