Weapon crop — the easy ways

You’re not missing something obvious. Hunting x, y in Aseprite’s status bar is awkward because the bar mixes file size, selection size, and cursor position depending on version and theme.

Most people do not ship games by reading that bar all day. They either use the engine’s region editor (Godot shows all four numbers clearly) or a one-click script in Aseprite.

Target file: assets/weapons/ranged/weapons.png → each gun’s numbers live on a WeaponData resource (resources/items/weapons/*.tres), not on the scratch scene. The scratch scene is only a ruler. Swapping guns = shop calls apply_from_weapon_data — no new game scene.

Gun “profiles” (where the data actually lives)

One WeaponData .tres = one gun profile (name, price, damage, fire rate, and how to crop the sprite).

  • Grid mode: turn off Use Sprite Region Rect on the resource; set Spritesheet Cell Index (good enough for many guns).
  • Exact crop mode: turn on Use Sprite Region Rect; paste x, y, w, h from the Godot scratch scene into Sprite Region Rect on that weapon’s .tres.
  • New gun: duplicate an existing .tres, rename, edit stats + crop, drag into ShopManager.available_weapons in scenes/world/Main.tscn if it should appear in the shop.

The player’s WeaponController → Starting Weapon picks the default profile for run start (same resource the shop can sell).

Scaling to many guns (you do not use WeaponController as the list)

WeaponController on the player is just the runtime hand holding one gun. The catalog of all guns is:

  1. One file per gun: resources/items/weapons/your_gun_name.tres (duplicate any existing .tres in that folder as a template).
  2. Sprite on each .tres: set Spritesheet Cell Index or Use Sprite Region Rect + paste Rect2(…) from the scratch tool.
  3. Shop pool: open scenes/world/Main.tscn → node ShopManager → Inspector → Available Weapons → set Size to how many guns you have, then drag each .tres into the slots. That array is “all guns the shop can roll.”
  4. Starting gun only: scenes/player/Player.tscn → WeaponController → Starting Weapon = whichever profile the player begins with (one reference, not forty).

Adding gun #41 = new .tres + one more entry in Available Weapons. You never paste forty crops into WeaponController.

What the four numbers mean

FieldMeaning
x, yTop-left corner of the crop inside the PNG (pixels from the image’s top-left).
w, hWidth and height of the crop in pixels.

Your sheet is 66 × 704. So x + w ≤ 66 and y + h ≤ 704.

Method 1 — All in Godot (recommended, ~2 minutes)

No Aseprite number hunting. Godot’s Inspector shows x, y, w, h after you drag the crop.

  1. In Godot, open scenes/debug/WeaponRegionScratch.tscn (included in this project).
  2. Select the Sprite2D node.
  3. In the Inspector, make sure Region Enabled is on (it should be).
  4. Click the Region Rect row — Godot opens a texture / region editor where you drag a rectangle on the image. Drag it around one gun.
  5. Close the editor. The Inspector now shows the four values under Region Rect: x, y, w, h.
  6. Fast copy: press F6 to run this scene. Use the on-screen Copy region to clipboard button (or F2). Clipboard gets a Rect2(…) line; if your crop is a perfect 33×32 cell, you also get a spritesheet_cell_index = N line for grid mode.
  7. Paste into the right WeaponData .tres: for a custom crop, enable Use Sprite Region Rect and paste into Sprite Region Rect. For grid mode, turn Use Sprite Region Rect off and type the cell index from the clipboard line.
  8. Open scenes/player/Player.tscn only if you’re testing the WeaponController node directly — normally you do not paste here; the .tres owns the crop.
  9. Tune Weapon Sprite Scale if the gun is too big/small on the knight.
This is the normal “how people do it” workflow: visual crop in the engine, numbers come from the Inspector.

Method 2 — Aseprite script (one click, all four numbers)

If you prefer to stay in Aseprite:

  1. Open weapons.png in Aseprite.
  2. M → drag marching ants around one gun.
  3. In Aseprite: File → Scripts → Open Script… (wording can be File → Scripts → Run depending on version).
  4. Choose this file: docs/aseprite_weapon_rect_from_selection.lua
  5. A dialog pops up with x, y, w, h and a Rect2(…) line. Copy those into the gun’s WeaponData .tres (Sprite Region Rect or grid index).

If the script errors, use Method 1 — Godot doesn’t depend on Aseprite’s Lua version.

Method 3 — Aseprite only (manual, slower)

  • w, h: with the selection active, Edit → Copy, then File → New — the new image’s canvas size is w and h.
  • x, y: use Godot Method 1 or the script instead of fighting the status bar. If you insist: the bar often shows the whole image as 66 704 — that is not your crop position. Move the cursor to the top-left pixel inside your selection and look for the moving cursor coordinates — those are x, y.

Copy values into the gun profile (not the scratch scene)

  1. Open the weapon’s resource under resources/items/weapons/ (or duplicate an existing .tres for a new gun).
  2. Enable Use Sprite Region Rect and set Sprite Region Rect to the same x, y, w, h from Method 1 or 2.
  3. For run start: scenes/player/Player.tscn → WeaponController → Starting Weapon = that .tres. For the shop: add the same resource to ShopManager → Available Weapons in scenes/world/Main.tscn.

You only touch WeaponController → Use Weapon Region Manual for one-off experiments; normal play uses WeaponData on each gun.

About “66 704 60 19” in Aseprite’s footer

Often 66 × 704 is simply the full file size. 60 × 19 can be your selection size (w, h). That still leaves x, y — use Method 1 or 2 so you’re not guessing.
