Enemy megapack rollout & wave plan

You have ~48+ character art kits under assets/sprites/characters/megapack/ (42 with Idle/000.png layout + 6 Goblin sheet folders). The game only runs four enemy scenes today. This document is the implementation plan: how to wire kits into scenes, drive pools by wave, scale HP with progression, and schedule mini-bosses — without blocking on “all 50 at once.”

Superseded for numbers + deferred backlog: use docs/Main_ENEMY_WAVE_ROADMAP_V2.html — see the table of contents link §11 · deferred (do later) for issues intentionally parked until after the wave spine ships.

Current code facts
  • WaveData exists with enemy_scenes, timing, and is_boss_wave — but WaveManager does not read it yet; it only shortens spawn interval.
  • EnemySpawner spawns a uniform random pick from one global enemy_scenes array on Main.tscn.
  • EnemyData defines health_scale_per_wave / speed_scale_per_wave — schema exists; EnemyAI is not shown applying wave scaling in the reviewed flow (wire-up is part of this plan).

1 — Target architecture (what we will build)

  1. Wave-driven spawner — WaveManager loads an ordered list of WaveData resources (or one WaveTable.tres holding an array). On wave_started, push the active wave’s enemy_scenes pool (and spawn interval / duration overrides) into EnemySpawner via a new method e.g. apply_wave(wave: WaveData, wave_index: int).
  2. One scene per “role” — Keep EnemyAI.gd + HealthSystem. Each new enemy is a duplicated Enemy.tscn (or a single generic Enemy_Megapack.tscn with an EnemyData export). Prefer EnemyData .tres per kit for tuning without duplicating scripts.
  3. Wave scaling hook — On spawn, pass current_wave (from WaveManager or a tiny autoload snapshot) into the enemy so HP/speed become: final_hp = round(base_hp * (1 + health_scale_per_wave * (wave - 1))) with a per-tier cap so late waves do not explode past fun.
  4. Mini-boss channel — Extend WaveData with exports such as mini_boss_scene: PackedScene, mini_boss_count: int, mini_boss_hp_multiplier: float, pause_trash_spawns: bool during intro optional. When the wave starts, spawn the mini-boss(es) once, then normal spawner fills trash.
  5. Full boss waves — Use existing is_boss_wave: longer duration, slower trash OR no trash, single boss_scene + adds phase later.

2 — Implementation phases (order of work)

Phase A — Spine (required)

  • Wire WaveManager to @export var waves: Array[WaveData] (or load path).
  • _start_wave(): resolve wave by index, apply duration/break to timers, call EnemySpawner.apply_wave(active).
  • Fallback: if array empty, keep today’s behavior so nothing breaks mid-migration.

Phase B — Scaling (required for “stronger over time”)

  • Either apply scaling in EnemyAI._ready from EnemyData + wave index, or add EnemyScaler.apply(enemy, wave).
  • Use health_scale_per_wave on data; add global_hp_curve on WaveData if some waves need extra +15% without new scenes.

Phase C — Content pipeline (parallelizable)

  • Batch-create scenes/enemies/megapack/ scenes: template + EnemyData per kit (name, base HP, speed tier).
  • Group kits by silhouette size (hitbox radius) so spawn ring does not clip walls.

Phase D — Boss / mini-boss (after A)

  • Implement mini-boss spawn from WaveData fields; mark spawned node add_to_group("mini_boss") for UI skull icon later.
  • Boss waves: test with FrogBoss or largest kit first.

3 — Enemy tier bands (megapack → role)

Tier F (tutorial trash): Dummy_LVL1–4, Mushroom — low HP, readable patterns.
Tier E (early swarm): Goblin Regular/Barrel variants not yet in scenes → fast/low or slow/tanky pairs.
Tier D (mid): Ent_LVL1–2, Monsterfly_01, Mimic_LVL1–2, Dwarfette_LVL1–2, FrogMonster.
Tier C (mid+): RhinoMonster_01_Regular, 02_Silver, Sorcerer_LVL1–2, Ent_LVL3, Dwarfette_LVL3.
Tier B (heavy): RhinoMonster_03_Gold, 05_Orc, 06_OrcRedSkinned, Sorcerer_LVL3–4, Mimic_LVL3–4, Ent_LVL4, MonsterSlasher_01.
Tier A (elite / mini-boss bodies): Cyclop_Archer_01 (already used), Vampire_Archer_01, RhinoMonster_04_Devil, 07–10 variants, Knight_LVL3–4.

Exact tier = tunable. Start conservative: if a sprite is large or sheet suggests boss, keep it out of random trash until wave 8+.

5 — HP progression (stronger enemies + scaling)

  • Per-kit base — Set in EnemyData.max_health (e.g. Dummy 12, Goblin green 30, Rhino 80).
  • Per-wave growth — hp_out = round(base * (1 + health_scale_per_wave * (wave - 1))); suggest 0.04–0.08 for trash, 0.02–0.04 for already huge kits.
  • WaveData override — Optional enemy_hp_multiplier: float = 1.0 for “this wave is brutal.”
  • Mini-boss — Multiply after wave scaling: hp * mini_boss_hp_multiplier (2.0 / 2.5 / …).
  • Cap — min(hp, base * 6) per non-boss to avoid one-shots breaking fun; bosses uncapped or higher cap.

6 — Mini-boss vs boss (definitions)

  • Mini-boss — Spawns at wave start, 1–2 count, larger HP multiplier, optional unique tint/scale 1.15×, drops bonus gold/XP later. Still uses EnemyAI until you add bespoke boss scripts.
  • Boss wave — is_boss_wave == true: clear trash spawns or cap at very low rate; spawn one boss_scene; extend wave_duration; UI banner “Boss!”

7 — Practical rollout (not all 50 day one)

  1. Wave 1–5 kit — Implement Phase A+B, then add 6–8 new scenes (Dummy, Mushroom, Ent1, Monsterfly, + keep current goblins).
  2. Mini wave 5 — Re-use Cyclop with multiplier first (fast win).
  3. Expand pool by band — Add 5–8 scenes per milestone until megapack coverage is complete.
  4. Automate (optional later) — Editor tool or script that reads megapack folder names and emits placeholder EnemyData + scenes.
